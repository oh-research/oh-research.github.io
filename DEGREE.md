# predict_korea_degree.html 작성 계획

## 목표

한국 연평균 기온 데이터(연도, 온도)를 좌표평면에 시각화하고, 추세선(선형 회귀)을 애니메이션으로 찾아서 연평균 기온이 15°C(아열대 기준)에 도달하는 연도를 예측한다.

## 데이터

| 연도 | 2000 | 2002 | 2004 | 2006 | 2008 | 2010 | 2012 | 2014 | 2016 | 2018 | 2020 | 2022 | 2024 |
|------|------|------|------|------|------|------|------|------|------|------|------|------|------|
| °C   | 11.9 | 11.9 | 12.4 | 12.4 | 12.3 | 11.7 | 11.6 | 12.6 | 13.0 | 12.2 | 12.6 | 12.4 | 14.0 |

- 2년 간격, 13개 데이터 포인트
- 선형 회귀: y = ax + b (a ≈ 0.0484, b ≈ -84.90)
- 15°C 도달 예측: (15 - b) / a ≈ 2064년

## 참고 파일

`fitting_lab.html`의 5단계(추세선 탐색 애니메이션)를 참고한다.

### fitting_lab.html에서 가져올 것
- **레이아웃 패턴**: 왼쪽 정보 패널 + 오른쪽 Canvas 영역
- **디자인 시스템**: CSS 변수(--bg, --panel, --accent 등), Jua/Noto Sans KR 폰트
- **Canvas 렌더링 패턴**: devicePixelRatio 대응, resize 처리, 좌표 변환 함수(xOf, yOf)
- **애니메이션 구조**: requestAnimationFrame 기반 phase 전환

### fitting_lab.html에서 바꿀 것
- 탭 구조 제거 → 단일 페이지
- 사용자 입력 슬라이더 제거 → 자동 애니메이션
- 좌표계: 수학 좌표(정수) → 실제 데이터 좌표(연도 1998~2070, 온도 10~16°C)

## 애니메이션 단계

### Phase 0: 초기 상태
- 데이터 포인트 13개를 좌표평면에 표시 (팝 애니메이션으로 하나씩 등장)
- 랜덤 초기 직선 표시 (최종 추세선과 충분히 다른 기울기/절편)
- 각 데이터 포인트와 직선 사이의 오차 막대(빨간색) 표시

### Phase 1: 추세선 수렴 (경사하강법 시뮬레이션)
- **핵심: a(기울기)와 b(절편)가 독립적으로 움직이며 수렴**
- 정규화된 좌표에서 경사하강법 + 모멘텀으로 경로를 사전 계산
  - x_norm = (year - MEAN_YEAR) / SD_YEAR 로 정규화하여 수치 안정성 확보
  - learning rate, momentum 파라미터로 오버슈팅 포함한 자연스러운 수렴 궤적
- 사전 계산된 경로(descentPath)를 따라 직선이 회전+이동하며 수렴
- 오차 막대가 실시간으로 줄어드는 것이 보임
- 오버레이에 현재 a, b, SSE 값 실시간 표시
- 수렴 완료 시 데이터 포인트에서 축하 파티클

### Phase 2: 추세선 연장
- 확정된 추세선이 데이터 범위(~2024년)에서 15°C 도달 지점까지 연장
- 연장 끝점에 파란색 이동 점 + 트레일 효과
- 15°C 교차점 도달 시 파티클 폭발 + 화면 흔들림
- 예측 연도 표시 (숫자 롤링 카운터 효과)

### 유의사항: 부드러운 수렴 움직임

경사하강법 시뮬레이션에서 이론적 움직임과 시각적 부드러움을 양립시킨다.

```
1. 정규화: 연도 값이 ~2000으로 크기 때문에 gradient가 불균형해진다.
   x_norm = (year - mean) / sd 로 정규화하면 a, b의 gradient 스케일이 비슷해져서
   두 변수가 독립적으로 의미있게 움직인다.

2. 모멘텀: 단순 GD는 직선적으로 수렴해서 시각적으로 밋밋하다.
   모멘텀을 추가하면 약간의 오버슈팅이 생겨 직선이 흔들리며 안착하는
   다이나믹한 움직임이 된다.

3. 시간 매핑: descentPath를 easeInOutCubic으로 재생하여
   초반 가속 → 중반 빠른 변화 → 후반 부드러운 안착 느낌을 준다.

4. 파라미터 튜닝 기준:
   - lr = 0.008 정도: 수렴이 200스텝 이내에 완료
   - momentum = 0.7 정도: 1~2회 오버슈팅 후 안착
   - 초기 직선: 최종과 기울기 부호가 반대이거나 2배 이상 차이나게 설정
   - 수렴 중 직선이 화면(TEMP 10~16°C) 안에 유지되는지 검증 필요
```

## 인터랙션: 데이터 포인트 ↔ 테이블 양방향 하이라이트

### 캔버스 점 클릭 → 테이블 하이라이트
- 클릭 좌표에서 가장 가까운 데이터 포인트 탐색 (25px 이내)
- 해당 점: 확대 + 펄스 링 + 연도/온도 라벨 표시
- 왼쪽 테이블의 대응 행에 `.highlight` 클래스 토글

### 테이블 행 클릭 → 캔버스 점 하이라이트
- `data-idx` 속성으로 행 ↔ 데이터 인덱스 매핑
- 동일한 하이라이트 상태(highlightIdx)를 공유
- 같은 항목 재클릭 시 하이라이트 해제

## Canvas 렌더링 주의사항

fitting_lab.html의 패턴을 따르되, 다음 사항을 준수한다.

```
1. ctx.save() / ctx.restore()로 drawScene 전체를 감싼다.
   - try/catch로 감싸서 예외 시에도 restore 보장

2. shadow 상태: shadowBlur=0 설정 시 반드시 shadowColor='transparent'도 함께 리셋

3. lineDash 상태: setLineDash([]) 전에 lineDashOffset=0 리셋
   - lineDashOffset은 globalTime * speed % dashTotal로 바운딩 (누적 방지)

4. globalAlpha: 설정 후 해당 draw 직후에 즉시 1로 리셋 (특히 파티클 루프 내부)

5. 애니메이션 생명주기:
   - 모든 setTimeout ID를 animTimeouts 배열에 저장
   - resetAll()에서 animTimeouts 전부 clearTimeout
   - animateIdle에서 animId===null 가드로 reset 후 재스케줄 방지
   - phase 전환 대기(setTimeout) 중에도 requestAnimationFrame 루프 유지 (화면 정지 방지)

6. resize 이벤트: 애니메이션 실행 중(animId !== null)이면 drawScene 호출 생략
   (다음 RAF 콜백이 새 크기로 그려줌)
```
